# 技术路线写作风格指南

> 基于16本NSFC成功申请书的组织逻辑提炼，禁止迁移具体研究内容。

## 一、技术路线的结构规范

### 标准结构（面上/青年/省基金）

```
3.2 研究方案 / 技术路线

[1段总体研究思路]

[总体技术路线图]

"3.1节所述各种研究方法贯穿整个技术路线的始终。"

[研究内容一的详细方案]
[研究内容一的技术路线图]

[研究内容二的详细方案]
[研究内容二的技术路线图]

[研究内容三的详细方案]
[研究内容三的技术路线图]

[可行性分析]
```

### 重点项目结构

```
3.3 技术路线

3.3.1 整体技术路线
[1段总体研究思路]
[总体技术路线图]

3.3.2 各个课题技术路线
[课题一技术路线]
[课题一技术路线图]

[课题二技术路线]
[课题二技术路线图]
  ...
```

## 二、技术路线图设计规范

### 图的要素

1. **数据源/输入**（左侧/上方）
2. **研究内容**（中间，按逻辑排列）
3. **技术手段/方法**（标注在研究内容旁或下方）
4. **预期成果/输出**（右侧/下方）
5. **逻辑关系**（箭头连接）

### 总体路线图布局模式

**模式A：左右布局**（最常见）
```
左侧：研究背景/数据源
中间：研究内容（纵向排列）
右侧：预期成果
方法标注在研究内容下方
```

**模式B：上下布局**
```
上方：研究背景/数据源
中间：研究内容（横向排列）
下方：预期成果
方法标注在研究内容下方
```

**模式C：树形布局**（重点项目常用）
```
根节点：项目名称
二级节点：研究内容/课题
三级节点：子研究内容
叶节点：研究方法
```

### 分内容路线图布局模式

```
上方：输入数据/前置条件
中间：处理流程（按步骤排列）
下方：输出结果/验证方式
侧边：使用的方法
```

### 路线图数量规范

| 项目类型 | 总体图 | 分内容图 | 合计 |
|---|---|---|---|
| 面上项目 | 1 | 3-4 | 4-5 |
| 重点项目 | 1 | 4-5 | 5-6 |
| 青年项目 | 1 | 2-3 | 3-4 |
| 省基金 | 1 | 3 | 4 |
| 人文社科 | 1 | 0 | 1 |

## 三、技术路线文字描述规范

### 总体研究思路段落

必须包含以下要素：
1. 研究背景/出发点
2. 研究内容之间的逻辑关系
3. 研究方法如何贯穿
4. 预期形成的研究闭环

标准表述模板：
```
本项目拟对[总体问题]进行深入研究。首先，基于[方法/技术]，[研究内容一]；
其次，基于[方法/技术]，[研究内容二]；然后，基于[理论/方法]，[研究内容三]；
最后，[整合/应用]，[总体目标]。其技术路线如图X所示。
```

### 分内容详细方案——展开层级选择

**默认采用细粒度展开**（按孙研究展开），除非用户显式指定粗粒度或其他方式。

| 展开模式 | 适用场景 | 决策规则 |
|---|---|---|
| **细粒度（默认）** | 子研究下有2-4个孙研究，孙研究间方法/逻辑有差异 | 默认选择，与研究内容的三级结构对齐 |
| **粗粒度** | 用户显式要求"按粗粒度写""整体写一段" | 仅在用户明确指定时使用 |

### 细粒度展开（默认）

技术路线按**子研究→孙研究**展开，与研究内容的三级结构严格对齐：

```
研究内容结构              技术路线结构（细粒度）
─────────              ──────────────────
总起大帽段        ←→     总体研究思路+总体路线图
  专题（子研究）    ←→     子研究帽段+子研究路线图
    孙研究内容      ←→     孙研究独立段落
```

#### 子研究帽段（4步逻辑）

每个子研究的技术路线开头写1段帽段，包含：

| 步骤 | 逻辑定位 | 典型表述 |
|---|---|---|
| 1 | 数据/平台基础 | "以XXX数据平台为基础" |
| 2 | 研究方法概述 | "采用XXX、XXX等研究方法" |
| 3 | 研究思路概括 | "从XXX出发"、"分别从XXX和XXX展开" |
| 4 | 引出孙研究 | "具体研究方案从以下X个方面展开" |

帽段模板：
```
[子研究名称]。以[数据/平台基础]，采用[方法1]、[方法2]等研究方法，
从[研究思路]出发，分别从[孙研究1]、[孙研究2]和[孙研究3]展开研究，
具体研究方案从以下X个方面展开，其技术路线如图X所示。
```

#### 孙研究段落（6步语义顺序）

每个孙研究写1段独立段落，按以下6步语义顺序展开：

| 步骤 | 逻辑定位 | 典型表述 | 出现频率 |
|---|---|---|---|
| 1 | **输入/前置条件**：基于什么数据/前置成果 | "基于3.2.1所述的..."、"在(1)中数据预处理后的..." | 100% |
| 2 | **问题定位**：本孙研究要解决什么具体问题 | "针对XXX问题"、"考虑到XXX" | 85% |
| 3 | **方法设计**：采用什么方法/技术 | "利用XXX方法"、"提出XXX模型" | 100% |
| 4 | **处理流程**：方法如何执行（首先→其次→然后→最后） | "首先...其次...然后...最后..." | 100% |
| 5 | **输出/结果**：产生什么结果/模型 | "实现XXX"、"构建XXX" | 90% |
| 6 | **衔接/价值**：与后续研究的衔接或价值 | "为后续XXX提供基础"、"为XXX提供依据" | 70% |

孙研究段落模板：
```
（N）[孙研究名称]

[输入/前置条件]，[问题定位]。[方法设计]，首先，[步骤1]；
其次，[步骤2]；然后，[步骤3]；最后，[步骤4]。[输出/结果]，
[衔接/价值]。
```

孙研究段落示例（抽象）：
```
（1）[数据预处理与特征提取]

基于[数据来源]，针对[数据质量问题/特征稀疏问题]，利用[方法名称]
对数据进行预处理。首先，对原始数据进行[清洗/筛选]；其次，利用[技术]
进行[特征提取/降维]；然后，基于[方法]进行[特征选择/融合]；最后，
构建[特征集/数据集]，为后续[孙研究2]的[建模/分析]提供数据基础。
```

### 粗粒度展开（仅在用户显式指定时使用）

技术路线按**子研究**展开，每个子研究写1段合并方案+1张图，孙研究内容合并在段落中。

#### 子研究段落（6步语义顺序）

| 步骤 | 逻辑定位 | 典型表述 |
|---|---|---|
| 1 | 输入/前置条件 | "基于XXX"、"在XXX的基础上" |
| 2 | 首先做什么 | "首先，对XXX进行..." |
| 3 | 其次做什么 | "其次，利用XXX方法..." |
| 4 | 然后/接着做什么 | "然后/接着，对XXX进行..." |
| 5 | 最后做什么 | "最后，将XXX应用于..." |
| 6 | 衔接下一子研究 | "为XXX提供基础" |

粗粒度段落模板：
```
[子研究名称]。[输入/前置条件]，首先，[步骤1]；其次，[步骤2]；
然后，[步骤3]；最后，[步骤4]。[衔接下一子研究]。
本专题研究方案的技术路线如图X所示。
```

### 可行性分析段落

可行性分析采用**维度菜单**模式，根据项目类型和用户提供的材料，从以下8个维度中选择展开：

| 维度 | 说明 | 适用场景 |
|---|---|---|
| 研究思路/逻辑可行 | 研究逻辑严谨、内容递进、技术路线符合系统工程原理 | 所有项目（必选） |
| 技术路线可行 | 技术手段先进、方案切实可行 | 所有项目（推荐） |
| 研究方法可行 | 方法成熟常用、负责人有方法功底 | 所有项目（推荐） |
| 研究基础/前期积累可行 | 已发表相关论文、已结题项目、在研项目支撑 | 有前期积累时（推荐） |
| 团队构成可行 | 团队结构合理、跨学科协同、人力充足 | 团队有特色时 |
| 数据/合作方可行 | 数据来源丰富、合作机构/企业提供支持 | 有合作方/数据来源时 |
| 平台/设备/条件可行 | 实验室、数据库系统、软件工具等 | 有平台条件时 |
| 实践/应用验证可行 | 应用场景可落地、临床/政策场景可验证 | 有应用场景时 |

**维度选择规范**：
- 面上/青年/省基金：选3-4个维度（研究思路+方法+前期积累，可选数据/合作方）
- 重点项目：选4-5个维度（研究基础+团队+数据+平台，可选实践验证）
- 用户提供了什么材料就重点写什么维度

**标准3段结构**（面上/青年/省基金，最常见）：
```
（1）本项目提出的研究思路清晰可行。[研究逻辑论述]。这构成了一个循序渐进、
由理论到应用的科学研究过程。管理理论、[学科]与信息科学技术紧密结合，
研究内容重点突出，研究方法先进实用，技术路线符合系统工程原理。

（2）本项目的研究方法是可行的。采用的主要有[方法列举]等方法，均为目前
[领域]研究中比较常用的方法，且项目负责人有扎实的研究方法功底。

（3）[根据用户材料选择维度：前期积累/数据合作/团队/平台等]。
```

**扩展结构**（有丰富前期积累或合作方时）：
```
（1）研究思路可行。[同上]。

（2）研究方法可行。[同上]。

（3）研究基础可行。[已发表相关论文X篇，已结题项目XXX，在研项目XXX，
这些前期研究为本项目奠定了坚实基础]。

（4）数据与合作方可行。[与XXX医院/机构/企业建立了合作关系，
XXX可为本项目提供数据/技术/平台支持]。
```

**重点项目结构**（4大块）：
```
3.X.1 研究基础可行
（1）理论研究的科学可行
（2）方法论的先进可行
（3）实践应用的验证可行

3.X.2 研究团队可行
（1）跨学科研究团队协同可行
（2）校企合作成果落地可行
（3）研究团队累积效应可行

3.X.3 研究数据可行
（1）数据来源丰富稳定
（2）数据融合通畅可行

3.X.4 工作条件可行
（1）研究平台完备且科研基础扎实
```

**结尾句**（所有结构通用）：
```
综上所述，该项目研究内容明确，研究方法先进实用，并具有结构合理的研究团队，
扎实的研究基础，齐备的研究设施，为本项目的顺利实施提供了强有力的保障。
```

## 四、技术路线与研究方法的对应规范

### 衔接句

方法部分与技术路线部分之间，使用标准衔接句：
```
3.1节所述各种研究方法贯穿整个技术路线的始终，构成本项目的研究方案，具体如下。
```

### 方法嵌入方式

**方式1（最常见）**：在路线图中标注"研究方法：XXX"，在文字中展开
```
研究内容一 ──→ 研究方法：文献分析；数据挖掘；数学建模
```

**方式2**：在路线图中用子节点展示方法
```
研究内容一
├── 方法1：XXX
├── 方法2：XXX
└── 方法3：XXX
```

**方式3（重点项目）**：在技术路线前单独写"核心研究方法"，然后在路线中引用

## 六、图文比例规范

- 文字描述比图更详细
- 图展示逻辑框架，文字展开具体步骤
- 每张图后必须有对应的文字说明
- 图的编号规范：图1（总体）、图2-N（分内容）

## 七、技术路线图设计建议

### 设计原则

1. 逻辑清晰：箭头方向一致，层次分明
2. 信息完整：包含数据源、方法、成果
3. 闭环设计：体现反馈和验证环节
4. 方法标注：在关键节点标注使用的方法

### Mermaid格式建议

总体路线图推荐使用 `graph TD`（自上而下）或 `graph LR`（自左而右）。

分内容路线图推荐使用 `graph TD`，并在子图中组织方法节点。

### 关键设计要素

- 研究内容节点用方框
- 方法节点用圆角框
- 数据/输入用平行四边形
- 成果/输出用菱形或加粗框
- 逻辑关系用实线箭头
- 反馈关系用虚线箭头
