# NSFC 文献综述写作风格指南

> 本指南从16本NSFC成功申请书的文献综述部分归纳而来，记录文献组织策略、综述与评述关系模式、批判性表达规律、文献服务NSFC逻辑方式。
> **核心原则**：文献综述的结构是从本子中"学习出来"的，不是"写死出来"的。本指南只归纳论证功能与组织逻辑，不提供固定句式模板。

---

## 一、文献综述的整体叙事逻辑

NSFC文献综述遵循"研究内容→文献主题→子主题→文献群→评述"的递进叙事：

1. **研究内容驱动主题划分**：每个研究内容对应一个文献主题（12/16本采用此方式）
2. **主题内分论点展开**：每个主题下分2-4个子主题，子主题按方法类型/研究视角/技术发展逻辑排列
3. **文献群构建**：子主题内将相关文献组织成群，形成对比或递进关系
4. **嵌入式评述**：每个主题段末以"综上所述"承担评述功能
5. **独立评述升华**：综述结束后以独立小节（如1.2.5）综合各主题空白，引出本项目

**叙事禁忌**：
- 禁止按时间线罗列文献
- 禁止文献摘要堆叠
- 禁止文献与研究主题脱节
- 禁止文献独立成体系（必须服务研究论证）

## 二、主题划分策略

### 策略1：按研究内容划分（最常见，12/16本）
- **适用场景**：研究内容明确、各内容相对独立的项目
- **组织方式**：每个研究内容对应一个文献主题
- **示例**：研究内容1→主题1.2.1，研究内容2→主题1.2.2，研究内容3→主题1.2.3，研究内容4→主题1.2.4

### 策略2：按方法类型划分（2/16本）
- **适用场景**：方法研究型项目，方法分类清晰
- **组织方式**：按方法类别组织文献
- **示例**：方法A→主题1.2.1，方法B→主题1.2.2，方法C→主题1.2.3，方法D→主题1.2.4

### 策略3：按研究视角划分（1/16本）
- **适用场景**：研究视角多元、需对比不同视角的项目
- **组织方式**：按研究视角组织文献
- **示例**：视角A→主题1.2.1，视角B→主题1.2.2

### 策略4：按检测/研究对象划分（1/16本）
- **适用场景**：检测/研究对象明确分类的项目
- **组织方式**：按检测对象组织文献
- **示例**：对象A→主题1.2.1，对象B→主题1.2.2，对象C→主题1.2.3

### 主题数量规范
| 项目类型 | 主题数量 | 子主题数量 |
|---|---|---|
| 面上项目 | 3-4个 | 每主题2-4个 |
| 重点项目 | 5个 | 每主题2-4个 |
| 青年项目 | 3个 | 每主题2-3个 |
| 省基金 | 3个 | 每主题2-3个 |
| 人文社科 | 简略（受2页限制） | 1-2个 |

## 三、段落组织规范（论证功能，非固定句式）

⚠️ 以下只标注段落的**论证功能**，不提供固定句式。具体表达由模型根据研究内容自适应生成。

### 主题段的论证功能骨架
1. **引入句**：说明本主题的研究范围和分论点
   - 功能：界定本主题讨论边界，预告分论点
2. **举例句**：引用具体文献，说明其研究内容
   - 功能：呈现现有研究成果，构建文献群
3. **对比句**：对比不同方法的优缺点
   - 功能：识别方法差异，为评述铺垫
4. **转折句**：指出现有研究的不足
   - 功能：嵌入批判性评述，识别研究空白
5. **总结句**：归纳本主题的研究进展和不足
   - 功能：以"综上所述"引出本主题的评述结论

### 文献排列逻辑（非时间顺序）
- **按方法类型排列**：同类方法聚集，便于对比
- **按研究视角排列**：不同视角递进
- **按技术发展排列**：从基础到前沿
- **按研究深度排列**：从理论到应用

## 四、文献引用格式

### 作者-年份制（优先，14/16本采用）
- **中文文献**：`姓名等(年份)`，如"张三等(2017)"
- **英文文献**：`Author等(年份)` 或 `Author et al.(年份)`，如"Yao (2009)"、"Yin 等 (2023)"
- **多文献引用**：按年份先后排列，用分号分隔

### 编号制（2/16本采用）
- 格式：`[1]、[2]、[3]`
- 按出现顺序编号
- 参考文献列表按编号排列

### 引用格式选择规则
- 默认采用作者-年份制
- 用户明确要求编号制时切换
- 全文引用格式必须一致

## 五、文献间连接词使用规律

从16本归纳的连接词功能分类（不固化模板，根据语境自适应使用）：

| 功能 | 连接词示例 |
|---|---|
| 举例引用 | 如、例如、譬如 |
| 补充引用 | 此外、另外、与此同时 |
| 转折指出不足 | 然而、但、与之不同的是、尽管如此 |
| 引出结论 | 因此、所以、综上、综上所述 |
| 引出后续研究 | 为此、鉴于此、基于此、有鉴于此 |
| 递进深化 | 进一步、更进一步、不仅如此 |
| 对比差异 | 相比之下、与之相对、不同的是 |

## 六、表格使用规范

### 使用频率
- 方法研究型/数据驱动型项目普遍使用文献表格
- 实证研究型、优化决策型项目通常不用文献表格
- 每个有文献表格的子主题通常1张表，方法研究型项目可达4-7张

### 文献表格的3种类型

#### 类型1：方法对比表
- **适用场景**：同一主题下有多种方法需对比时
- **列结构**：分类（可选）/ 文献 / 方法或研究内容 / 评述
- **功能**：按方法分类对比，每行有批判性评述
- **评述列要求**：不是文献摘要，而是**一句话批判性评价**（如"解决非单调准则处理难题，奠定价值函数基础"、"评估效果显著优于现行评分标准"）

#### 类型2：应用汇总表
- **适用场景**：需展示方法在不同场景/疾病/领域中的应用时
- **列结构**：分类 / 应用场景或疾病 / 方法 / 目标（可选）/ 评述 / 文献
- **功能**：按应用场景汇总，体现方法的应用广度和时效性
- **命名策略**：常用"近5年XXX相关研究"限定时效性
- **评述列要求**：简短评价该方法在该场景中的效果或局限

#### 类型3：特征矩阵表
- **适用场景**：需对比多个研究覆盖的特征/维度/场景时
- **列结构**：文献 / 多个特征维度列 / 用√/×标记覆盖情况
- **功能**：一目了然展示哪些研究覆盖了哪些维度，识别空白
- **评述方式**：通过√/×矩阵本身隐含评述（空白维度即研究空白），表格后文字总结指出缺失维度

### 表格位置规范

#### 位置模式A：综述→表格→评述
- 先文字综述该子主题的文献
- 再用表格汇总对比
- 最后1段文字总结表格内容并指出不足
- 适用于：文献较多、需先铺垫再对比的子主题

#### 位置模式B：引入→表格→评述
- 先1-2句引入该子主题
- 直接用表格汇总
- 最后1段文字总结表格内容并指出不足
- 适用于：文献以方法对比为主、文字综述已在前面段落完成的子主题

### 表格使用判断
- 文献数量≥5篇且需对比时使用
- 方法类型≥3种时使用
- 表格不能替代文字评述，必须配套
- 非文献表格（概念分类表、知识框架表）也可在文献综述中使用，用于整理概念体系

## 七、综述→评述的过渡方式

### 方式1：嵌入式评述（6/16本）
- 每个主题段末以"综上所述"承担评述功能
- 适用于：研究内容紧凑、评述点明确的项目

### 方式2：独立评述小节（10/16本）
- 综述结束后以独立小节（如1.2.5或1.2.4）综合评述
- 适用于：研究内容复杂、需综合多个空白的项目

### 过渡技巧
- 综述中嵌入评述（如"然而，此类方法通常缺乏较好的可解释性"）
- 独立评述开头概括各主题的研究价值
- 独立评述末尾引出本项目研究方向

---

## 八、批判性表达规律（不固化模板）

### 批判性表达的3种功能
1. **指出现有研究的局限**
   - 功能：识别方法/数据/模型的不足
   - 表达特征：指出"多关注...，对...处理还缺乏相应的研究"

2. **识别研究空白**
   - 功能：明确尚未涉及的研究领域
   - 表达特征：指出"现有研究对...涉及很少"

3. **引出本项目研究方向**
   - 功能：建立空白→本项目的论证链
   - 表达特征：以"因此，本项目拟..."引出

### 批判性表达的禁忌
- 禁止无依据的否定
- 禁止情绪化批评
- 禁止脱离研究主题的批判
- 批判必须服务于研究论证，不是为批判而批判

## 十、文献服务NSFC逻辑方式

### 1. 文献→研究问题支撑
- 文献综述按研究内容组织（12/16本）
- 每个研究内容对应一个文献主题
- 文献评述指出每个主题的研究空白
- 研究空白直接对应本项目的研究问题

### 2. 文献→方法合理性支撑
- 文献综述中介绍各种方法
- 指出现有方法的局限
- 本项目方法是对现有方法的改进或创新
- 技术路线部分可引用"如综述部分所见"衔接

### 3. 文献→创新合理性支撑
- 文献评述指出研究空白
- 本项目的创新点直接对应研究空白
- 形成"空白→创新"的论证链
- 创新特色部分呼应文献评述的结论

## 十一、与已有技能的协同边界

- `fund-background-writer`：负责立项依据的 **1.1 研究意义** 部分
- **本技能**：负责立项依据的 **1.2 国内外研究现状及发展动态分析 + 文献评述** 部分
- `fund-research-content-writer`：负责「研究目标—研究内容—关键科学问题」
- `fund-technical-route-writer`：负责「研究方法—技术路线—创新和特色」

本技能不覆盖其他技能的职责范围，但需读取用户研究构想（研究目标、研究内容、关键科学问题、研究方法、创新特色）以确保文献服务研究论证。
