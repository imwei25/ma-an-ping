# NSFC 写作信息表（模板）

> 用途：本技能在写立项依据研究意义部分前，需要这份信息表来锚定背景与后续研究内容的逻辑对应关系。

**默认采用理论创新导向**：优先关注科学问题/假说的可证伪性、理论贡献的清晰性、验证维度的完备性（详见 `theoretical_innovation_guidelines.md`）

## 输入来源

用户调用本技能时，**至少**会提供以下材料：

1. 【必填】**研究内容**：专题/子课题的划分及各专题的研究要点
2. 【必填】**研究方法**：各专题拟采用的关键方法和技术
3. 【必填】**技术路线**：研究的整体步骤和逻辑流程
4. 【必填】**特色与创新**：2-4 条创新点
5. 【必填】**研究目标**：2-4 条目标
6. 【必填】**关键科学问题**：1-3 条疑问句式的问题

如果以上输入不足，**必须先追问补全**再继续。

## 锚定映射

收到输入后，按以下规则建立"输入→背景"的逻辑锚定（详见 SKILL.md Step 3）：

| 后续输入 | 锚定到背景的段落 | 映射规则 |
|---------|----------------|---------|
| 研究内容的专题数 | 段落C的困境条数 | 每个专题对应一个现实困境 |
| 研究方法的技术挑战 | 段落D的不足条数 | 每个挑战对应一个现有不足 |
| 技术路线的步骤 | 段落E的方案展开顺序 | 方案 = 技术路线的浓缩叙述 |
| 特色与创新的条目 | 理论意义的条目 | 创新点 → 学术化表达 |
| 研究目标的条目 | 实践意义的条目 | 目标 → 应用化表达 |
| 关键科学问题 | 段落F的科学问题列举 | 可直接引用或改写 |

**核心原则**：锚定是逻辑层面的"影子对应"，不是文字层面的"复制粘贴"。背景有背景自己的视角和语言。

## 可选补充（按需）

- **研究对象/应用场景**：一句话边界（人群/疾病/材料/系统/任务）
- **核心科学假设**：1 句可证伪表述（陈述句，预测性结果，不写验证方式）
- **前期基础（可核验）**：论文/专利/数据/原型/预实验现象
- **字数限制**：例如目标 4000 字（中文字符）与允许偏差 ±200
- **术语口径**：研究对象如何称呼、核心指标名称、缩写首次出现是否要全称等
