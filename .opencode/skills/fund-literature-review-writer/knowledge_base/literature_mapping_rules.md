# NSFC 文献→研究问题/方法/创新映射规则

> 本文件记录文献如何映射到NSFC研究论证结构的三个关键环节：研究问题、方法合理性、创新合理性。
> **核心原则**：文献不是独立成体系，而是服务用户研究构想。每篇文献都应在论证链中找到位置。

---

## 一、文献→研究问题映射规则

### 核心映射逻辑
```
用户研究内容 → 文献综述主题 → 文献主题评述 → 研究空白 → 用户研究问题
```

### 映射规则

#### 规则1：研究内容→文献主题对应
- 每个用户研究内容对应一个文献主题
- 文献主题的数量与研究内容数量一致（或略多）
- 文献主题的命名应体现研究内容的核心问题

#### 规则2：文献主题→研究空白识别
- 每个文献主题的评述必须指出研究空白
- 研究空白应具体、可识别，避免泛泛而谈
- 研究空白应与用户研究内容直接相关

#### 规则3：研究空白→研究问题对应
- 研究空白直接对应用户的研究问题
- 每个研究问题都应有文献空白支撑
- 形成"空白→问题"的对应关系

### 映射模式（从16本归纳）

#### 模式1：直接对应（最常见）
- 文献主题1的空白 → 研究问题1
- 文献主题2的空白 → 研究问题2
- 文献主题3的空白 → 研究问题3
- **适用**：研究内容明确、各内容独立的项目

#### 模式2：综合对应
- 多个文献主题的空白综合 → 关键科学问题
- **适用**：研究内容复杂、需综合多个空白的项目

#### 模式3：递进对应
- 文献主题1的空白 → 研究问题1（基础）
- 文献主题2的空白 → 研究问题2（进阶）
- 文献主题3的空白 → 研究问题3（应用）
- **适用**：研究内容有递进关系的项目

### 映射验证清单
- [ ] 每个研究内容是否有对应的文献主题？
- [ ] 每个文献主题是否指出了研究空白？
- [ ] 每个研究空白是否对应用户的研究问题？
- [ ] 研究问题是否都有文献空白支撑？

## 二、文献→方法合理性映射规则

### 核心映射逻辑
```
文献综述介绍方法 → 评述指出方法局限 → 本项目方法改进/创新 → 技术路线衔接
```

### 映射规则

#### 规则1：文献综述介绍现有方法
- 文献综述中应介绍与用户研究方法相关的现有方法
- 方法介绍应包括：方法原理、适用场景、主要贡献
- 方法介绍应按逻辑顺序组织（非时间顺序）

#### 规则2：评述指出方法局限
- 评述应指出现有方法的局限
- 局限应具体、可识别，避免泛泛而谈
- 局限应与用户研究方法的改进点对应

#### 规则3：本项目方法改进/创新
- 本项目方法是对现有方法的改进或创新
- 改进点应直接对应现有方法的局限
- 创新点应有文献支撑（不是凭空产生）

#### 规则4：技术路线衔接
- 技术路线部分可引用"如综述部分所见"衔接
- 方法的合理性在文献综述中已建立基础
- 技术路线是对文献综述方法介绍的深化

### 映射模式（从16本归纳）

#### 模式1：方法改进型
- 现有方法A有局限X → 本项目方法A'改进了局限X
- **适用**：方法研究型、数据驱动型

#### 模式2：方法融合型
- 现有方法A有局限X，现有方法B有局限Y → 本项目融合A和B，克服X和Y
- **适用**：方法研究型、机制研究型

#### 模式3：方法创新型
- 现有方法均无法解决某问题 → 本项目提出全新方法
- **适用**：前沿研究型项目

### 映射验证清单
- [ ] 文献综述是否介绍了与用户方法相关的现有方法？
- [ ] 评述是否指出了现有方法的局限？
- [ ] 用户方法的改进点是否对应现有方法的局限？
- [ ] 技术路线是否与文献综述衔接？

## 三、文献→创新合理性映射规则

### 核心映射逻辑
```
文献评述指出研究空白 → 本项目创新点对应空白 → 形成"空白→创新"论证链
```

### 映射规则

#### 规则1：文献评述指出研究空白
- 文献评述应综合各主题的研究空白
- 研究空白应具体、可识别
- 研究空白应与用户创新点对应

#### 规则2：本项目创新点对应空白
- 每个创新点都应对应一个研究空白
- 创新点不是凭空产生，而是填补空白
- 形成"空白→创新"的论证链

#### 规则3：创新特色部分呼应文献评述
- 创新特色部分应呼应文献评述的结论
- 创新点的合理性在文献评述中已建立基础
- 创新特色是对文献评述的回应

### 映射模式

#### 模式1：空白填补型
- 空白X → 创新点1填补空白X
- 空白Y → 创新点2填补空白Y
- **适用**：研究内容明确的项目


#### 模式2：方法突破型
- 现有方法局限 → 创新点突破方法局限
- **适用**：方法研究型项目

#### 模式3：视角拓展型
- 现有研究视角单一 → 创新点拓展研究视角
- **适用**：实证研究型、优化决策型

### 映射验证清单
- [ ] 文献评述是否指出了研究空白？
- [ ] 每个创新点是否对应一个研究空白？
- [ ] 创新特色部分是否呼应文献评述？
- [ ] "空白→创新"论证链是否完整？

## 四、与用户研究构想的对齐机制

### 对齐维度

#### 维度1：用户研究目标 → 文献综述主题划分
- 文献综述的主题划分应服务用户研究目标
- 主题数量、主题内容应与研究目标匹配
- 主题划分应体现研究目标的核心问题

#### 维度2：用户研究内容 → 文献主题对应
- 每个研究内容对应一个文献主题
- 文献主题的内容应覆盖研究内容的相关文献
- 文献主题的评述应指出与研究内容相关的研究空白

#### 维度3：用户关键科学问题 → 文献评述的研究空白
- 文献评述的研究空白应对应用户的关键科学问题
- 每个关键科学问题都应有文献空白支撑
- 文献评述应引导到用户的关键科学问题

#### 维度4：用户研究方法 → 文献中的方法介绍和局限
- 文献综述应介绍与用户方法相关的现有方法
- 评述应指出与用户方法相关的现有方法局限
- 用户方法的改进点应对应现有方法的局限

#### 维度5：用户创新特色 → 文献评述的空白对应
- 文献评述的空白应对应用户的创新特色
- 每个创新特色都应有文献空白支撑
- 创新特色的合理性在文献评述中已建立基础

### 对齐验证流程

```
Step 1: 读取用户研究构想（研究目标、研究内容、关键科学问题、研究方法、创新特色）
    ↓
Step 2: 根据研究内容划分文献主题
    ↓
Step 3: 检索文献，构建文献群
    ↓
Step 4: 撰写文献综述，每个主题对应一个研究内容
    ↓
Step 5: 撰写文献评述，指出每个主题的研究空白
    ↓
Step 6: 验证研究空白→研究问题/方法/创新的对应关系
    ↓
Step 7: 调整文献综述和评述，确保对齐
```

### 对齐失败的处理
- 如果文献无法支撑某个研究问题：提示用户补充文献或调整研究问题
- 如果文献评述的空白与用户创新点不对应：调整文献评述或提示用户调整创新点
- 如果文献综述与研究内容脱节：重新组织文献综述

## 五、文献服务的禁忌

### 禁忌1：文献独立成体系
- 文献综述不能脱离用户研究构想独立组织
- 每篇文献都应在论证链中找到位置
- 文献综述的目的是服务研究论证，不是展示文献阅读量

### 禁忌2：文献与研究主题脱节
- 文献必须与研究主题相关
- 不能为了凑数量而引用无关文献
- 文献的组织应服务研究主题

### 禁忌3：文献评述只是总结
- 文献评述不能只是文献摘要的堆叠
- 文献评述必须包含批判性判断
- 文献评述必须指出研究空白

### 禁忌4：文献→研究问题/方法/创新的断裂
- 文献综述、文献评述、研究问题、方法、创新之间必须有逻辑联系
- 不能出现文献综述与后续研究脱节的情况
- 每个环节都应服务整体研究论证

---

## 六、映射规则的执行优先级

1. **研究内容对齐优先**：文献综述的主题划分必须对齐用户研究内容
2. **研究空白识别优先**：文献评述必须识别与用户研究问题相关的空白
3. **方法局限指出优先**：文献评述必须指出与用户方法相关的现有方法局限
4. **创新点对应优先**：文献评述的空白必须对应用户的创新点

当出现冲突时，按以上优先级处理。
