# 研究方法写作风格指南

> 基于16本NSFC成功申请书的组织逻辑提炼，禁止迁移具体研究内容。

## 一、组织方式选择

### 方式A：按方法类型列举（推荐，12/16本采用）

适用：面上项目、青年项目、省基金

结构：
```
3.1 研究方法
（1）文献研究法
（2）数据挖掘/机器学习方法
（3）数学建模法
（4）决策分析/优化方法
（5）实证研究法
```

方法数量规范：
- 面上项目：5-8种
- 青年项目：5-6种
- 省基金：5-7种

### 方式B：按研究内容/专题组织

适用：研究内容差异大、方法组合不同的项目

结构：
```
3.1 研究方法
3.1.1 专题一：[名称]的研究方法
  - 子研究1的方法
  - 子研究2的方法
3.1.2 专题二：[名称]的研究方法
  ...
```

### 方式C：按课题组织

适用：重点项目或研究内容逻辑独立的项目

结构：
```
3.2 核心研究方法
（1）"课题一：XXX"的核心研究方法
（2）"课题二：XXX"的核心研究方法
  ...
```

每个课题需说明：核心挑战 → 针对挑战的方法 → 方法如何解决挑战

### 方式D：简略列举

适用：人文社科等格式受限项目

结构：一句话列举方法名称，详细内容合并到技术路线中。

## 二、方法段落结构

### 标准段落模板（方式A）

```
（N）[方法名称]

[一句话说明方法用途]。[在本项目中如何使用]。[（可选）方法的创新性或改进点]。
```

### 段落要素优先级

1. **必须**：方法名称 + 在本项目中做什么
2. **推荐**：解决什么问题 / 为什么采用这种方法
3. **可选**：方法的创新性或改进点

### 描述深度规范

- 说明"用什么方法解决什么问题"，而非详细算法描述
- 需要说明方法选择的理由
- 如有方法创新，需简要说明创新点
- 避免过度展开技术细节（留给技术路线部分）

## 三、方法与研究内容的映射规范

### 核心原则

方法与研究内容是**多对多**关系：
- 一个方法可支撑多个研究内容
- 一个研究内容可使用多个方法

### 映射表述方式

在方法段落中，需明确说明该方法支撑哪些研究内容：

```
[方法名称]。[用途说明]，用于支撑研究内容[一/二/三]中[具体任务]的研究。
```

### 方法间协同关系

需体现方法之间的输入输出关系：
```
利用[方法A]对数据进行[处理]，为[方法B]提供[输入]。
```

## 四、常见方法类型及表述规范

| 方法类型 | 标准表述模板 | 出现频率 |
|---|---|---|
| 文献研究法 | "通过查阅国内外关于[领域]的文献，分析并提炼最新理论研究成果" | 100% |
| 数据挖掘方法 | "对[数据]进行加工、整理及预处理，基于[技术]研究[问题]" | 88% |
| 数学建模法 | "根据收集到的[数据/理论]，构建[模型]，[解决什么问题]" | 81% |
| 机器学习/深度学习 | "基于[算法]，构建[模型]，实现[目标]" | 69% |
| 优化方法 | "利用[优化算法]对[模型]进行求解，[达到什么效果]" | 50% |
| 实证研究法 | "通过和[合作方]合作，将理论成果用于[应用]，验证[有效性]" | 38% |
| 专家访谈/问卷调查 | "采用[访谈/问卷]，[获取什么信息/验证什么]" | 63% |
| 多准则决策方法 | "采用[决策方法]，评估/分析[问题]" | 31% |

## 五、字数规范

- 面上项目：500-800字
- 重点项目：800-1500字（按课题展开）
- 青年项目：400-600字
- 省基金：400-700字

## 六、方法组织的递进规律

- 面上/青年/省基金：方法列举 → 技术路线 → 详细方案（方法与技术路线分离）
- 重点项目：核心方法 → 技术路线 → 详细方案（方法更深入，按课题组织）
- 人文社科：方法简述 → 技术路线（合并，受格式限制）

## 七、衔接句规范

方法部分与技术路线部分之间，使用标准衔接句：
```
3.1节所述各种研究方法贯穿整个技术路线的始终。
```
