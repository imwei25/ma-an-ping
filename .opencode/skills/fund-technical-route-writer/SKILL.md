# NSFC「研究方法—技术路线—创新和特色」写作技能

## 核心定位

本技能**不是**创造新的研究方案，而是在用户已有研究构想基础上，对研究方法、技术路线和创新特色进行**学术化表达、逻辑优化与规范化呈现**。

## 技能范围

本技能只负责 NSFC 申请书中的研究方法、技术路线与创新特色部分。

## 核心原则

1. **不创造研究方案**：在用户已有研究构想基础上进行学术化表达
2. **逻辑优先**：优先保证研究逻辑的严谨性，而非追求复杂表述或华丽措辞
3. **闭环设计**：研究内容→关键科学问题→研究方法→技术路线→创新特色必须形成完整闭环
4. **用户优先**：优先保留用户提供的材料、方法和框架
5. **禁止迁移**：不将历史本子的具体研究内容、研究对象、数据来源和结论迁移到新项目

## 工作流程

### Step 1：阅读用户输入

阅读用户提供的研究目标、研究内容、关键科学问题及所有附加材料。

**必填**：研究目标、研究内容、关键科学问题
**选填**：指定研究方法、已有草稿、参考代码、算法框架、方法设计文档、流程图草稿等

### Step 2：分析研究逻辑

分析研究目标→研究内容→关键科学问题的逻辑链，识别：
- 研究内容之间的逻辑关系（递进/并列/支撑）
- 项目类型（数据驱动/优化决策/方法研究/实证研究/机制研究）
- 需要哪些类型的方法

### Step 3：设计或优化研究方法

根据 `methodology_style_guide.md` 和 `methodology_patterns.md`：

**情况1**：用户仅提供研究目标、研究内容和关键科学问题
→ 自行设计研究方法，按方法选择决策树和项目类型选择方法组合

**情况2**：用户已提供研究方法
→ 优先理解和继承用户的方法设计，检查方法与研究内容的映射是否完整

**情况3**：用户已提供部分研究方法和部分技术路线
→ 重点优化其逻辑衔接，优先保留用户已有设计

输出：研究方法部分（含方法列举、用途说明、与研究内容的映射）

### Step 4：构建技术路线

根据 `technical_route_style_guide.md` 和 `technical_route_patterns.md`：

1. **确定展开层级**（默认细粒度）：
   - 细粒度（默认）：按子研究→孙研究展开，与研究内容三级结构对齐
   - 粗粒度：仅在用户显式指定时使用
2. 设计总体技术路线（1段总体研究思路 + 1张总体路线图）
3. 设计分内容技术路线：
   - 细粒度：每个子研究写帽段+路线图+孙研究独立段落（6步语义顺序）
   - 粗粒度：每个子研究写1段合并方案+1张图
4. 确保方法嵌入路线、形成闭环

输出：技术路线文字描述 + 技术路线图设计方案

### Step 5：设计技术路线图

提供技术路线图的 Mermaid 代码或设计建议：
- 总体路线图：体现全局逻辑
- 分内容路线图：体现具体操作
- 标注研究方法
- 体现闭环设计

### Step 6：提炼创新和特色

根据 `innovation_style_guide.md` 和 `innovation_patterns.md`：

1. 从研究内容中识别突破点
2. 从关键科学问题中识别创新对应
3. 从研究方法中识别方法创新
4. 按创新类型（方法/视角/理论/应用/框架/集成）组织创新点
5. 确保创新点与研究内容、关键科学问题呼应

输出：创新和特色部分

### Step 7：闭环验证

根据 `validation_menu.md` 和 `anti_patterns.md`：

1. 检查研究内容→研究方法→技术路线→创新特色闭环
2. 检查方法与内容一致性
3. 检查技术路线与正文一致性
4. 检查创新点与内容一致性
5. 检查反模式
6. 检查格式规范

## 适配机制

### 用户仅提供基本信息

自行设计研究方法与技术路线，按知识库中的范式组织。

### 用户已提供研究方法

优先理解和继承用户的方法设计。如有遗漏，补充建议而非替换。

### 用户已提供部分材料

重点优化逻辑衔接，不推翻重建。优先使用用户指定的方法和框架。

## 输出格式

输出包含以下部分：

1. **研究方法**：按知识库风格撰写的方法部分
2. **技术路线文字描述**：总体思路 + 分内容详细方案
3. **技术路线图设计方案**：Mermaid代码或设计建议
4. **创新和特色**：按知识库风格撰写的创新特色部分

## 语言约束

- 输出语言与用户输入语言一致
- 学术化表达，避免口语化
- 优先保证逻辑严谨性，而非追求复杂表述
- 方法名称使用中英文对照（首次出现时）
