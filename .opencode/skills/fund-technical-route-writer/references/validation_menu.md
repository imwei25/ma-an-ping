# 验证菜单

## 一、闭环检查

### 检查1：研究内容→研究方法→技术路线→创新特色闭环

- [ ] 每个研究内容是否有对应的研究方法？
- [ ] 每个研究方法是否在技术路线中体现？
- [ ] 每个创新点是否能在研究内容中找到落点？
- [ ] 研究方法→技术路线→创新特色是否形成完整闭环？

### 检查2：研究目标→研究内容→关键科学问题闭环

- [ ] 研究方法是否响应研究目标？
- [ ] 研究方法是否解决关键科学问题？
- [ ] 创新特色是否与关键科学问题呼应？

### 检查3：输入→处理→输出闭环

- [ ] 技术路线中每个模块是否有明确的输入？
- [ ] 每个模块是否有明确的输出？
- [ ] 模块之间是否有输入输出衔接？
- [ ] 整体是否形成反馈闭环？

## 二、一致性检查

### 检查4：方法与内容一致性

- [ ] 方法数量是否与研究内容匹配？
- [ ] 方法类型是否与研究内容性质匹配？
- [ ] 方法描述是否与研究内容对应？

### 检查5：技术路线与正文一致性

- [ ] 技术路线图是否与文字描述一致？
- [ ] 总体路线图是否与分内容路线图一致？
- [ ] 路线图中的方法标注是否与方法部分一致？

### 检查6：创新点与内容一致性

- [ ] 每个创新点是否在研究内容中有落点？
- [ ] 创新点数量是否与研究内容数量匹配？
- [ ] 创新点之间是否有逻辑重叠？

## 三、反模式检查

### 检查7：方法反模式

- [ ] 是否存在方法堆砌（列举方法但未说明用途）？
- [ ] 是否存在方法与研究内容脱节？
- [ ] 是否存在方法选择缺乏理由？

### 检查8：技术路线反模式

- [ ] 技术路线是否只是方法列举的重复？
- [ ] 技术路线是否缺乏逻辑衔接？
- [ ] 技术路线是否缺乏闭环设计？

### 检查9：创新特色反模式

- [ ] 创新点是否只是口号，缺乏具体论证？
- [ ] 创新点是否使用了绝对化表述（无依据的"首次""领先"）？
- [ ] 创新点之间是否有逻辑重叠？
- [ ] 创新点数量是否过多（>5个）或过少（<2个）？

## 四、格式检查

### 检查10：字数规范

- [ ] 研究方法字数是否符合规范？
- [ ] 技术路线字数是否符合规范？
- [ ] 创新特色字数是否符合规范？

### 检查11：图表规范

- [ ] 技术路线图数量是否符合规范？
- [ ] 图的编号是否正确？
- [ ] 每张图后是否有对应的文字说明？

## 五、质量检查

### 检查12：逻辑严谨性

- [ ] 研究逻辑是否严谨（而非追求复杂表述）？
- [ ] 方法选择是否有充分理由？
- [ ] 技术路线是否可执行？
- [ ] 创新点是否真实来源于研究内容？

### 检查13：风格一致性

- [ ] 是否与历史成功本子的写作风格一致？
- [ ] 方法描述深度是否适当？
- [ ] 创新点论证是否充分？
