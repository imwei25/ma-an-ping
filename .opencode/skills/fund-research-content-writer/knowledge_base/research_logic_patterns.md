# NSFC 研究方案逻辑范式（面上项目）

> 本文件重点总结研究方案的**组织逻辑**，而非固定模板。
> 核心理解："这一部分为什么存在"，而不仅仅是"这一部分怎么写"。
> 禁止将历史本子的具体研究内容、研究对象、数据来源和结论迁移到新项目。
> 本技能只覆盖：研究目标、研究内容、关键科学问题。

---

## 一、研究目标展开逻辑

### 1.1 目标推导路径

```
立项依据中的研究缺口 → 核心问题提炼 → 研究目标设计
```

**为什么这样推导**：
- 目标必须从立项依据的缺口直接推导，否则目标与依据脱节
- 每个目标回答一个缺口，形成"问题→目标"的因果链
- 目标之间形成递进或互补，确保研究方案的完整性

**推导验证**：
- 检查每个目标是否都能追溯到立项依据中的某个缺口
- 检查所有缺口是否都有对应的研究目标
- 检查目标之间是否有逻辑关系（不能是简单并列）

### 1.2 目标表述范式

**基本格式**：动词 + 研究对象 + 预期成果

**常见动词选择**：
| 动词 | 适用场景 | 含义 |
|---|---|---|
| 构建 | 理论/模型/框架 | 从无到有建立 |
| 揭示 | 规律/机理/机制 | 发现未知 |
| 提出 | 方法/策略/方案 | 创新性给出 |
| 建立 | 体系/指标/评估 | 系统性搭建 |
| 开发 | 工具/平台/系统 | 工程性实现 |
| 探索 | 关系/路径/模式 | 开放性研究 |
| 设计 | 算法/机制/流程 | 规划性工作 |

### 1.3 目标间逻辑关系

**递进关系**（最常见）：
- 目标1（基础/理论）→ 目标2（核心/方法）→ 目标3（应用/验证）
- 前者输出 = 后者输入

**互补关系**：
- 目标1（宏观层面）+ 目标2（微观层面）+ 目标3（整合层面）
- 各目标从不同角度回答同一核心问题

**因果链关系**：
- 目标1（原因分析）→ 目标2（影响机制）→ 目标3（干预策略）
- 体现"为什么→怎么样→怎么办"的逻辑

## 二、研究内容拆解逻辑（三级结构）

### 2.1 三级结构总览

```
研究内容
├── 总起大帽段（整体引入）
├── 专题一（子研究内容）
│   ├── 专题帽段
│   ├── （1）孙研究内容1
│   ├── （2）孙研究内容2
│   └── （3）孙研究内容3
├── 专题二
│   ├── 专题帽段
│   ├── （1）...
│   ├── （2）...
│   └── （3）...
└── 专题三
    ├── 专题帽段
    ├── （1）...
    ├── （2）...
    └── （3）...
```

**为什么需要三级结构**：
- 总起大帽段：让评审快速理解整体逻辑
- 专题帽段：让评审理解每个专题的内部逻辑
- 孙研究内容：展开具体的论证

### 2.2 总起大帽段逻辑

**功能**：对整个研究内容进行整体性引入

**逻辑结构**：
1. 研究背景/定位（1-2句）
2. 核心研究思路（1句）
3. 为什么现在做（新数据/新方法/新需求等时机要素，1句）
4. 3-4个专题的标题和逻辑关系（核心内容）
5. 专题间的递进关系说明

**为什么需要总起大帽段**：
- 评审在30秒内需要理解你的研究设计
- 没有总起段，评审需要读完所有专题才能理解逻辑
- 总起段是"地图"，让评审带着整体框架去读细节

### 2.3 专题帽段逻辑

**功能**：说明本专题的研究思路和内部逻辑

**逻辑结构**：
1. 本专题要解决什么问题（1-2句）
2. 拟采用什么研究思路（1句）
3. 孙研究内容之间的逻辑关系（"首先...其次...最后..."）
4. 本专题的独立贡献（即使前置专题未完成，本专题仍有独立价值，1句）

**为什么需要专题帽段**：
- 让评审在进入孙内容之前理解专题的整体设计
- 避免孙内容变成简单的并列列表
- 专题帽段是专题的"小地图"

### 2.4 专题间递进关系

**三种递进模式**：

#### 模式A：递进式（最常见）

```
专题1（基础/理论）→ 专题2（核心/方法）→ 专题3（应用/验证）
```

- 前者输出 = 后者输入
- 适用于：机制研究型、优化决策型

#### 模式B：并列汇聚式

```
专题1（维度A）+ 专题2（维度B）+ 专题3（维度C）→ 统一方案
```

- 各专题从不同维度研究同一核心问题
- 适用于：数据驱动型

#### 模式C：层层深入式

```
专题1（现象/效果）→ 专题2（机制/因素）→ 专题3（影响/应用）
```

- 逐步深入理解
- 适用于：预测分析型、社科类

### 2.5 递进逻辑验证

**删除法**：删除任一专题，其余内容是否还能成立？
- 如果能 → 递进关系不够紧密
- 如果不能 → 递进关系成立

**逆向追溯法**：从最后一个专题开始，逆向追溯其依赖关系
- 每个专题是否都依赖前一个专题的输出？
- 是否存在"孤岛"专题？

## 三、科学问题凝练逻辑

### 3.1 凝练路径

```
专题 → 提取核心挑战 → 上升为科学问题
```

**为什么需要凝练**：
- 科学问题是专题的"灵魂"，体现研究的科学价值
- 问题层次高于专题，不是简单重复专题标题
- 每个问题通常横跨1-2个专题，体现问题的综合性

### 3.2 问题与专题的关系

**错误做法**：每个专题对应一个科学问题（1:1映射太机械）

**正确做法**：
- 科学问题横跨1-2个专题
- 2-3个科学问题覆盖3-4个专题
- 问题体现"科学性"而非"技术性"

**科学性 vs 技术性**：
| 维度 | 科学问题 | 技术问题 |
|---|---|---|
| 关注点 | 为什么/机理/规律 | 怎么做/实现/操作 |
| 层次 | 理论层面 | 工程层面 |
| 表述 | "如何揭示...机理" | "如何实现...功能" |
| 价值 | 推动学科发展 | 解决具体应用 |

### 3.3 问题表述范式

**四种常见范式**：

1. **测度/刻画型**："如何测度/刻画...的关系/特征"
2. **机理/机制型**："...的内在机理/驱动机制是什么"
3. **演化/规律型**："...的演化规律/时空分异规律是什么"
4. **优化/决策型**："...的优化机理/协同逻辑是什么"

### 3.4 问题论证结构

每个科学问题必须包含三部分：

```
1. 问题陈述：清晰表述科学问题
2. 为什么难：现有研究的不足/无法直接解决的原因
3. 拟如何突破：简述突破思路（不需要详细方案）
```

**"为什么难"的论证逻辑**：
- 现有方法只考虑了A，忽略了B
- 现有理论适用于C情境，但不适用于D情境
- 现有研究在E层面有成果，但在F层面尚属空白

## 四、完整闭环验证

### 4.1 闭环检查清单

- [ ] 每个研究目标有对应的专题（1:1映射）
- [ ] 关键科学问题覆盖所有专题
- [ ] 专题之间有明确递进关系
- [ ] 研究目标→研究内容→关键科学问题形成完整闭环
- [ ] 术语跨章节一致
- [ ] 无孤立的专题或孙研究内容

### 4.2 映射关系图

```
研究目标          专题（子研究内容）    科学问题
  (1) ─────────→ 专题1 ─────────→ 问题1
  (2) ─────────→ 专题2 ─────────→ 问题2
  (3) ─────────→ 专题3 ─────────→ 问题3（可选）
                  ↕                ↕
               递进关系          横跨关系
```

**映射验证**：
- 从左到右：每个目标都有专题和问题
- 从右到左：每个问题都能追溯到具体专题和目标
- 无断链：不存在只有目标没有专题，或只有专题没有问题的情况

### 4.3 一致性检查

**术语一致性**：
- 研究对象/缩写/指标口径跨章节一致
- 首次出现给出缩写与定义
- 全文缩写保持一致

**逻辑一致性**：
- 研究目标 → 专题：1:1映射
- 专题 → 关键科学问题：多对多覆盖
- 无孤立的专题（没有问题覆盖的专题）
